How This Hub Works

The hub does not contain the games. It launches separate HTML game files and reads their progress keys from browser storage.

Use one repository and one web origin. The hub and games must be published together for browser storage to be shared reliably.

Faculty configure the hub in one place. Edit HUB_CONFIG and the GAMES array in the JavaScript section. Each game must use matching storage keys.

Ordered play is optional. When ordered locking is enabled, a game unlocks after the previous game is completed. When disabled, students may open any game.

Local progress is not a gradebook. It is a convenience layer. Use telemetry or a required submission method when completion must be formally verified.
